Третья нейронка (2 правки приняты):

Четвёртая нейронка (10 правок приняты):

## ВИРТУАЛЬНЫЙ P&L — ОБРАТНАЯ СВЯЗЬ БЕЗ РОБОТОВ

Каждый сгенерированный сигнал содержит:
- entry_price — цена на момент сигнала
- stop_loss — через ATR
- target — через ATR
- ttl_minutes — динамический по слою детектора
- detector_layer — MICRO / MESO / MACRO
- direction = LONG или SHORT
- wavefunction_state = ACCUMULATE / DISTRIBUTE / HOLD
- top_detector = какой детектор доминировал
- state = ACTIVE

Динамический TTL:
```
TTL_MAP = {
  MICRO: 60,    // минут — GRAVITON, DARKMATTER, DECOHERENCE, HAWKING
  MESO: 240,    // 4 часа — ACCRETOR, PREDATOR, ENTANGLE
  MACRO: 1440   // 1 день — ATTRACTOR, WAVEFUNCTION, CIPHER
};
const ttl = TTL_MAP[detectorLayer[top_detector]] || 240;
```

Фоновый процесс каждые 5 минут проверяет все ACTIVE сигналы:
- Для LONG: если max(price за TTL) >= target → WIN; если min(price) <= stop → LOSS; иначе → EXPIRED
- Для SHORT — зеркально

Верификация через MFE/MAE: max_favorable_excursion >= target AND max_adverse_excursion <= stop внутри TTL

### Обратная связь для WAVEFUNCTION
- LONG по ACCUMULATE → WIN → +0.01 к ACC→LONG, -0.01 от ACC→HOLD
- LONG по DISTRIBUTE → LOSS → -0.01 от DIST→LONG, +0.01 к DIST→HOLD
- Только при win_rate < 40% на выборке 30+ сигналов
- Transition matrix остаётся стохастической (сумма строк = 1, все ≥ 0)

### Обратная связь для BSCI весов (weekly)
- win_rate по каждому top_detector
- Детектор с win_rate > 60% → увеличить вес
- Детектор с win_rate < 40% → уменьшить вес
- Isolated activation оценка: для чистой оценки конкретного детектора — смотреть случаи, когда его score был высок, а другие низки, и проверять движение цены

## ДЕТЕКТОРЫ — ФИНАЛЬНЫЕ ФОРМУЛЫ v4.2

### 1. GRAVITON — Гравитационное линзирование стакана

Обрезка стакана:
- cutoffLevel = min level where cumulative_volume >= 0.8 * totalSideVolume
- Все расчёты ТОЛЬКО на уровнях [0..cutoffLevel]
- Min уровней (v4.2): if cutoffLevels < 3 → use all levels (100% объёма)
- Пустая сторона (v4.2): if bid_depth == 0 || ask_depth == 0 → graviton_score = 0, reason: "incomplete_ob"

center_mass_bid = Σ(volume_i × price_i) / (Σvolume_i + ε) для bid side [0..cutoffLevel]
center_mass_ask = Σ(volume_i × price_i) / (Σvolume_i + ε) для ask side [0..cutoffLevel]
ε = 1e-6

separation = (center_mass_ask - center_mass_bid) / (mid_price + ε)

ATR-нормализация separation для сравнимости между тикерами:
- separation_norm = separation / (ATR_pct + ε)
- ATR_pct = ATR(14) / mid_price

asymmetry_norm = asymmetry_raw / (current_spread + ε) — безразмерная
asymmetry_raw = (Σ(bid_vol × dist_from_cm_bid) - Σ(ask_vol × dist_from_cm_ask)) / (total_vol + ε)
dist_from_cm = |price - center_mass|

graviton_score = sigmoid(α × separation_norm + β × |asymmetry_norm|)
α = 2.0, β = 1.5 (калибруются)

detect_walls():
- wall = уровень где volume > 3 × median_volume_per_level
- wall_proximity = min(distance_to_wall) / max(spread, 0.001 × mid_price)
- wall_score = Σ(wall_volume × w_depth_k) / (total_side_volume + ε)
- w_depth_k = exp(-depth_k / (median_depth + ε)) — median вместо avg для устойчивости
- Интегрируется в graviton_score как дополнительный сигнал

Явная нормализация score ∈ [0, 1]: graviton_score = min(1, max(0, raw_score))
Fallback: if (!orderbook || depth < 3) → graviton_score = 0, reason: "no_ob_data"

ПРИОРИТЕТ: П2

### 2. DARKMATTER — Тёмная материя (айсберги)

Entropy по 80% cutoff levels — стабильная глубина:
- Используем cutoffLevel из GRAVITON (переиспользование кода)
- observed_entropy = Shannon_entropy(объёмы по уровням [0..cutoffLevel])
- expected_entropy:
  - Медиана за последние 5 ОС-сессий (v4.2 — быстрее адаптируется к смене режима)
  - Только данные ОС (10:00-16:50 МСК) для baseline
  - v1 fallback (первые дни, < 5 сессий): expected_entropy ≈ log2(avg_cutoff_depth) × 0.85

- darkmatter_entropy_score:
  - ΔH_norm = (expected_entropy - observed_entropy) / (expected_entropy + ε)
  - observed >= expected → score = 0
  - observed < expected → score = ΔH_norm ∈ (0, 1]

iceberg_score:
- Группируем сделки по price_level
- Consecutive runs одинакового объёма (ПОДРЯД!)
- n_consecutive ≥ 3
- MIN_ICEBERG_VOLUME = max(0.005 × dailyTurnover, 10 × median_trade_size)
- iceberg_score_at_level = n_consecutive_same_vol / n_total_at_level
- weight = exp(-distance_from_best / (avg_depth + ε)) — НЕ 1/(1+dist)
- iceberg_score = weighted_average(iceberg_score_at_level)

Защита при тонком стакане: if cutoff_depth < 5 → darkmatter_entropy_score = 0

darkmatter_score = 0.5 × darkmatter_entropy_score + 0.5 × iceberg_score
Явная нормализация score ∈ [0, 1]

ПРИОРИТЕТ: П1

### 3. ACCRETOR — Аккреционный диск (дробление заявок)

Фильтруем сделки: trade_value < 0.3 × median_trade_value
trade_value = volume × price (денежное выражение для универсальности)

Нормировка признаков перед DBSCAN:
```
const scaled = trades.map(t => [
  (t.time - t0) / eps_time, // нормировка на 60 сек
  (t.price - p0) / tick_size // нормировка на 1 tick
]);
const clusters = dbscan(scaled, eps=1.0, minSamples=5);
```
- eps_time = 60 секунд, eps_price через нормировку = 1 tick (безразмерный)
- min_samples = 5
- Окно: 200 последних сделок
- Пересчёт: каждые 30 секунд

Min trades guard (v4.2): if n_trades < 30 → accretor_score = 0, reason: "insufficient_trades"

accretor_score = (n_clustered_trades / (n_small_trades + ε)) × cluster_concentration
cluster_concentration = avg_cluster_size / (ATR(14) / (tick_size + ε))
- 60% мелких сделок кластеризовано → крупный игрок дробит заявку
- НЕ дублирует DECOHERENCE: DECOHERENCE = pattern в объёмах, ACCRETOR = spatial clustering

Явная нормализация score ∈ [0, 1]

ПРИОРИТЕТ: П2

### 4. DECOHERENCE — Декогеренция (потеря когерентности)

Символьный поток:
- if (price_change > 0) symbol = round(log2(volume)) × +1
- else if (price_change < 0) symbol = round(log2(volume)) × -1
- else symbol = round(log2(volume)) × sign(tick_rule_direction)
- Clip: symbol = sign(dir) × min(round(log2(volume)), 10) — явный clip в [-10, +10]
- Алфавит: от -10 до +10 (21 символ, включая 0)
- Скользящее окно W=100 сделок → частотное распределение

Shannon entropy:
- H = -Σ(p_i × log2(p_i)) для всех p_i > 0

Miller-Madow коррекция:
- H_corrected = H_observed + (S_observed - 1) / (2 × W × ln(2))
- S_observed = количество уникальных символов в окне

H_max с floor:
- effective_H_max = max(log2(activeSymbols), log2(7))
- // Минимум 7 символов — ниже недостаточно данных

Декогерентность = 1 - (H_corrected / effective_H_max)

Alphabet guard (v4.2): if activeSymbols < 5 → decoherence_score = 0, reason: "insufficient_alphabet"
Low activity guard: if price_change_count / W < 0.3 → decoherence_score = 0, reason: "low_activity"

Явная нормализация score ∈ [0, 1]

ПРИОРИТЕТ: П1

### 5. HAWKING — Излучение Хокинга (периодичность алго)

Ресэмплинг в равномерную сетку перед FFT/Welch:
- Бины по 100 мс (binMs = 100)
- Значение бина = количество сделок в бине
- Преобразует точечный процесс в равномерный временной ряд

Минимальное окно:
- n_trades < 50 → hawking_score = 0
- 50 ≤ n_trades < 100 → сырой FFT на ресэмплированном ряде
- n_trades >= 100 → Welch's method

Autocorrelation lag 1..20 на ресэмплированном ряде
Периодичность = max(|ACF(k)|) для k=2..20

noise_ratio = 1 - (peak_power / (median_psd × bandwidth + ε))
- bandwidth = частотный диапазон анализа

Адаптивная зона алгоритмов:
- avg_freq = n_trades / duration_sec
- algo_zone = [0.1 × avg_freq, 3 × avg_freq]

hawking_score = периодичность × (1 - noise_ratio)
Убрать ВСЕ упоминания WVD
Явная нормализация score ∈ [0, 1]

ПРИОРИТЕТ: П1

### 6. PREDATOR — Хищник (охота на стопы)

Основной детектор — aggression_ratio + cumulative delta acceleration

5-фазный цикл:
- STALK — цена приближается к уровню скопления стопов
  - Триггер: |current_price - estimated_stop_level| < 1.5 × ATR(14)
  - timeout = 30 мин → сброс в IDLE
- HERDING — мелкие сделки толпы на уровне
  - Триггер: n_small_trades_at_level / n_total_trades > 0.6 AND avg_trade_size < 0.3 × median_trade_value
  - timeout = 15 мин → сброс в IDLE
- ATTACK — агрессивный пробой
  - Триггер: aggression_ratio > 2.0 AND |price_change| > 2 × ATR(14) AND cumDelta_acceleration > 0
  - timeout = 5 мин → FALSE_BREAKOUT
- CONSUME — кит выкупает (цена возвращается, CumDelta +)
- FALSE_BREAKOUT — новостной пробой (цена не возвращается, CumDelta -)

CONSUME vs FALSE_BREAKOUT:
- После ATTACK в течение window_confirm:
  - window_confirm = 3 × ATR / (price × 0.001) минут, bounds [2, 10]
  - price_reversion = (current_price - attack_extreme) / max(|pre_attack_price - attack_extreme|, 0.5 × tick_size)
  - ATR-нормированный reversion threshold (v4.2):
    - reversion_threshold = max(0.3, min(0.7, 0.5 × (1 - ATR_pct / 0.05)))
    - При высокой волатильности → порог ниже (быстрее подтверждаем)
    - При низкой → порог выше (строже)
  - delta_flip через FLOW (z-scored с fallback):
    - if (n_flow_observations >= 20):
      - zDelta = (delta_flow - mean_flow) / (std_flow + ε)
      - delta_flip = Math.abs(zDelta) > 1.5 && sign(delta_flow_after) !== sign(delta_flow_during)
    - else:
      - delta_flip = sign(delta_flow_after) !== sign(delta_flow_during)
    - FLOW = ΔcumDelta за период, НЕ LEVEL (кумулятивное)
  - if (price_reversion >= reversion_threshold && delta_flip) → CONSUME → LONG/SHORT
  - if (price_reversion < reversion_threshold || !delta_flip) → FALSE_BREAKOUT → AWAIT

estimated_stops(level):
- 0.35 × volume_cluster_density(level)
- 0.25 × round_number_bonus(level) = 1 если кратен 5/10 пунктам
- 0.25 × recent_breakout_frequency(level)
- 0.15 × vwap_distance_penalty(level)
- Начальные коэффициенты, калибруются

Явная нормализация score ∈ [0, 1]

ПРИОРИТЕТ: П2 (estimated_stops + FALSE_BREAKOUT), П3 (volume POC для стопов)

### 7. CIPHER — Шифр (камуфляж алгоритма)

УРОВЕНЬ 1 (каждую секунду):
- features = [volume, trade_size, interval] за окно W
- ОБЯЗАТЕЛЬНО: robustNormalize(features, window=100) — median/IQR
- Без нормализации volume на 3 порядка больше interval → PCA видит только объём
- PCA(n_components=3).fit(features)
- dominance_ratio = explained_variance_ratio_[0]
- Если dominance_ratio > 0.6 → один компонент доминирует → алгоритм
- cipher_quick = dominance_ratio

УРОВЕНЬ 2 (каждые 30 сек, гистерезис: старт при >0.5, стоп при <0.4):
- cov_condition = conditionNumber(covariance(features))
- if (cov_condition > 1000) → skip ICA, cipher_score = cipher_quick
- ICA на normalized matrix, max_iterations=200, fixed seed
- Если ICA не сошлась → cipher_score = cipher_quick
- kurtosis = mean(|IC_i|⁴) / mean(|IC_i|²)²
- Baseline-коррекция kurtosis:
  - excess_kurtosis = kurtosis - 3
  - if excess_kurtosis > 2 × MAD(kurtosis_history) → несколько алгоритмов
  - // Финансовые данные ВСЕГДА лептокуртичны — абсолютный порог >3 некорректен
- cipher_deep = (cipher_quick + kurtosis_normalized) / 2

Финальный cipher_score:
- cipher_quick ≤ 0.5 → score = cipher_quick
- cipher_quick > 0.5 → score = cipher_deep
- ICA fallback → score = cipher_quick

Явная нормализация score ∈ [0, 1]

ПРИОРИТЕТ: П2

### 8. ENTANGLE — Запутанность (кросс-активные корреляции)

v1:
- Двойной тест стационарности ADF + KPSS:
```
function ensureStationarity(series: number[]): number[] {
  const adfResult = augmentedDickeyFullerTest(series);
  const kpssResult = kpssTest(series);
  // ADF отвергает + KPSS не отвергает → стационарно
  if (adfResult.pvalue < 0.05 && kpssResult.pvalue > 0.05) return series;
  // Дифференцируем
  const diffed = series.slice(1).map((v, i) => v - series[i]);
  const adfDiff = augmentedDickeyFullerTest(diffed);
  const kpssDiff = kpssTest(diffed);
  if (adfDiff.pvalue < 0.05 && kpssDiff.pvalue > 0.05) return diffed;
  return []; // нестационарно даже после разностей → пропустить
}
```
- Применяем к bid/ask volume flows (первые разности cumVolume)
- Granger с лагом = 3 (фиксированный v1)
- v1: только intra-ticker (bid→ask, ask→bid)
- Bonferroni поправка: p_threshold = 0.05 / N_tests

v2 (П3): AIC/BIC для топ-20 пар + Hilbert transform

Явная нормализация score ∈ [0, 1]

ПРИОРИТЕТ: П2

### 9. WAVEFUNCTION — Волновая функция (байесовский интент)

v1 (фиксированная матрица + ресэмплинг + log-weights + jitter):

Transition matrix (фиксированная):
```
[0.7, 0.2, 0.1] ACCUMULATE → {ACCUMULATE, DISTRIBUTE, HOLD}
[0.2, 0.6, 0.2] DISTRIBUTE → {ACCUMULATE, DISTRIBUTE, HOLD}
[0.1, 0.2, 0.7] HOLD → {ACCUMULATE, DISTRIBUTE, HOLD}
```

Observation model (КРИТИЧЕСКИ — без этого PF невозможно):
```
// Параметры likelihood per state:
ACCUMULATE: μ = [+0.3, +0.2, +0.2], ν = 5
DISTRIBUTE: μ = [-0.3, -0.2, -0.2], ν = 5
HOLD: μ = [ 0.0, 0.0, 0.0], ν = 4

// Observation vector:
z = [cumDelta_norm, ofi_norm, trade_imbalance_norm]

// Student-t likelihood (робастная):
L(z | state) = StudentT(z; μ_state, Σ_state, ν_state)
Σ_state — диагональная ковариация, оценивается из rolling window
```

Начальное распределение: равномерное (1/3 каждое состояние)

Мониторинг вырождения + ресэмплинг:
```
N_eff = 1 / (Σ(weights²) + ε)
if (N_eff < 0.5 × n_particles) {
  particles = systematicResample(particles, weights);
  weights = new Array(n_particles).fill(1 / n_particles);
}
```

Post-resampling jitter (КРИТИЧЕСКИ — без этого PF коллапсирует):
```
σ_jitter = K × range / N^(1/d)
particles.forEach(p => {
  p.params = p.params.map(v => v + gaussianRandom(0, σ_jitter));
});
// K = 0.05, d = dimensionality
```

Логарифмирование весов:
```
log_weights[i] += log(likelihood + ε)
// Нормализация через log-sum-exp:
max_log_w = max(log_weights)
log_sum = max_log_w + log(Σexp(log_weights - max_log_w) + ε)
log_weights = log_weights.map(w => w - log_sum)
// Извлечение: weight = exp(log_weight)
```

v1: фиксированные 200 частиц
v2: адаптивные 200-1000, только для топ-30 тикеров по обороту

H_posterior = -Σ(p_state × log2(p_state)) для posterior over {ACC, DIST, HOLD}
n_particles = floor(200 × (1 + H_posterior_normalized × 4))

v1.5: Обратная связь по P&L (см. Virtual P&L)
v2: EM-алгоритм, минимум 500+ сигналов

Явная нормализация score ∈ [0, 1]: probability of dominant state

ПРИОРИТЕТ: П2 (обязательно), v1.5/v2 = П3

### 10. ATTRACTOR — Аттрактор (смена режима)

Takens на detrended prices (КРИТИЧЕСКИ):
```
const detrended = prices.map((p, i) => p - SMA(prices, 20)[i]);
const embedded = takensEmbed(detrended, d=3, tau=autoTau);
// НЕ сырые цены (random walk → ложная геометрия)
// НЕ log-returns (теряют информацию об уровне)
```

takens_convergence:
- d = 3 фиксированный, FNN для d — v2
- Авто τ: τ = findFirstZeroACF(detrended_series) || 5
- Пересчитывается РАЗ В ЧАС, min=2, max=20
- Silverman robust bandwidth:
  - h = 1.06 × min(σ, IQR/1.34) × N^(-1/5)
- KDE: ρ(Y) = (1/N) × Σ K((Y - Y_i) / h)
- Takens_convergence = концентрация плотности вокруг аттрактора

volume_profile_attractor:
- POC = уровень с максимальным объёмом
- Прилипание к POC: |price-POC| < 2 ticks > 60% времени
- POC distance guard: if |POC - current_price| > 1 × ATR → volume_profile_score = 0

price_stickiness:
- sticky = |price[t] - price[t-1]| < 0.5 × EMA(spread, 10)
- stickiness_ratio = sticky_time / window_length

attractor_score = 0.4 × takens + 0.3 × volume_profile + 0.3 × stickiness
Regime change trigger: attractor_center_shift > 1 × ATR OR stickiness drops from >0.6 to <0.3

Явная нормализация score ∈ [0, 1]

ПРИОРИТЕТ: П2

### 11. BSCI — Композитный индекс

w_k(t) = w_k(t-1) + η × (S_k(t) - w_k(t-1)) × w_k(t-1)
η = 0.03
min_w = 0.04
Нормализация: Σw_k = 1

ДВА УРОВНЯ АДАПТАЦИИ (документировано):
- Intraday (η-адаптация): реагирует на СИЛУ СИГНАЛА. S_k = «сила», не «активность».
- Weekly (P&L корректировка): реагирует на ТОЧНОСТЬ НАПРАВЛЕНИЯ. Ортогональный механизм.

Adaptive daily weight decay (v4.2):
```
if (RVI < 0.3) {
  // Спокойный рынок — мягкий decay
  w_k = 0.99 × w_k + 0.01 × (1/K) // half-life 69 дней
} else {
  // Волатильный рынок — агрессивный decay
  w_k = 0.97 × w_k + 0.03 × (1/K) // half-life 23 дня
}
RVI = Relative Volatility Index
```

L2-регуляризация ОТКЛОНЕНА
Accuracy decay ОТКЛОНЁН

Мультиколлинеарность penalty:
if avg_pairwise_correlation > 0.7 → BSCI_adjusted = BSCI × (1 - 0.3 × avg_correlation)

```
function robustNormalize(features: number[], window: number = 100): number[] {
  const slice = features.slice(-window);
  const med = median(slice);
  const iqr = quantile(slice, 0.75) - quantile(slice, 0.25) + ε;
  return features.map(v => (v - med) / iqr);
}
```

## ФОРМУЛЫ CONFIDENCE И ПОРОГИ

Confidence formula:
```
confidence = (25 × BSCI + 25 × conv + 15 × rsi + 15 × robots + div_raw) / 100

Где:
  BSCI ∈ [0,1]     — уже есть в системе
  conv ∈ [0,1]     — convergence = n_detectors_above_0.5 / n_total
  rsi ∈ [0,1]      — |RSI(14) - 50| / 50
  robots ∈ [0,1]   — robotVolumePct / 100
  div_raw ∈ [-10, 20] — clamp(20 × (topDet - 0.5) / 0.35, -10, 20)

confidence ∈ [-0.1, 1.0] → clampScore → [0, 1]
```

Н1: Confidence capped на 0.80 без divergence — это ДИЗАЙН:
```
max_base = (25×1 + 25×1 + 15×1 + 15×1) / 100 = 0.80
// confidence = 1.0 требует conviction от top-детектора (divergence)
// confidence = 0.8 = все системы согласны, но нет выдающегося лидера
```

Signal direction: direction = sign(Σ w_k × dir_k × score_k)

Пороги сигналов:
- Тип: LONG / SHORT / AWAIT / BREAKOUT
- Условие: BSCI >= 0.55 AND convergence >= 7 AND topDetector >= 0.75
- TTL: MICRO=60м, MESO=240м, MACRO=1440м
- Entry / Stop / Targets: через ATR(14)
