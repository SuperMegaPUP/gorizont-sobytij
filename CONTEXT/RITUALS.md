# 📜 РИТУАЛЫ

> 8 обязательных ритуалов — предотвращение отклонения от архитектуры
> Версия документа: 1.0.0

---

## 🚀 РИТУАЛ 1: Начало сессии

**Когда:** Каждый раз при старте новой сессии.

**Порядок:**
1. Прочитать `/CONTEXT/CONTEXT.md`
2. Прочитать `/CONTEXT/VERSIONING.md`
3. Прочитать `/CONTEXT/FEATURES.md`
4. Прочитать `/CONTEXT/HISTORY.md` (последние 3 записи)
5. Прочитать `/CONTEXT/ARCHITECTURE.md`
6. Прочитать `/CONTEXT/RITUALS.md`
7. Прочитать `/CONTEXT/WORKLOG.md`
8. Уточнить у пользователя текущую задачу и приоритет

**Цель:** Полное погружение в контекст, никаких действий вслепую.

---

## 🔄 РИТУАЛ 2: Конец сессии

**Когда:** Перед завершением работы.

**Порядок:**
1. Обновить `/CONTEXT/HISTORY.md` — что делали, что изменили
2. Обновить `/CONTEXT/FEATURES.md` — если менялся статус фич
3. Обновить `/CONTEXT/WORKLOG.md` — лог общения
4. Обновить `/CONTEXT/CONTEXT.md` — если требуется коррекция
5. Обновить `/CONTEXT/VERSIONING.md` — если менялась версия
6. Проверить, что все изменения задокументированы
7. Сообщить пользователю итоги сессии

**Цель:** Ничего не потерять между сессиями.

---

## ✅ РИТУАЛ 3: Pre-commit

**Когда:** Перед каждым git commit.

**Порядок:**
1. `npm run lint` — проходит?
2. `npm run test:ci` — проходит?
3. Проверить, что не закоммичены секреты (env, токены)
4. Сообщение коммита по convention: `type(scope): description`
5. Если коммит большой — разбить на логические части

**Цель:** Чистая история, рабочий код.

---

## 🚀 РИТУАЛ 4: Pre-deploy

**Когда:** Перед каждым деплоем.

**Порядок:**
1. Все тесты проходят
2. Линт чистый
3. `npx next build` — успешен
4. Проверить `.vercel/project.json` — правильный проект
5. Деплой LAB → smoke-test
6. Деплой PROD → smoke-test
7. Обновить `HISTORY.md` и `WORKLOG.md`

**Цель:** Рабочий деплой, никаких сюрпризов.

---

## 📊 РИТУАЛ 5: Обновление фич

**Когда:** После реализации/изменения фичи.

**Порядок:**
1. Найти фичу в `FEATURES.md`
2. Обновить статус: `pending` → `in_progress` → `done`
3. Добавить ссылку на коммит/PR
4. Обновить прогресс %
5. Если фича завершена — добавить дату завершения

**Цель:** Актуальная картина прогресса.

---

## 🧪 РИТУАЛ 6: Обновление тестов

**Когда:** После изменения детектора/формулы.

**Порядок:**
1. Обновить unit-тесты под новую формулу
2. Добавить синтетический тест если новый сценарий
3. Проверить, что старые тесты не сломаны
4. Запустить `npm run test:ci`
5. Обновить `HISTORY.md` — какие тесты изменены

**Цель:** Регрессии не пройдут.

---

## 📈 РИТУАЛ 7: Еженедельный обзор

**Когда:** Раз в неделю (понедельник, начало ОС).

**Порядок:**
1. Прочитать `FEATURES.md` — что в работе, что отложено
2. Прочитать `HISTORY.md` — что было на прошлой неделе
3. Обновить приоритеты в `FEATURES.md`
4. Проверить BSCI-direction correlation (если данные есть)
5. Замерить распределения BSCI/convergence на 100 тикерах
6. Обновить `CONTEXT.md` — текущий фокус

**Цель:** Стратегическое планирование, не застревать.

---

## 🔒 РИТУАЛ 8: Безопасность

**Когда:** Раз в спринт или при подозрении.

**Порядок:**
1. Проверить `.env*` файлы — не закоммичены?
2. Проверить логи — нет токенов в console.log?
3. Проверить CI/CD — токены в secrets, не в коде?
4. Проверить Redis/DB — доступ только через env?
5. Обновить `VERSIONING.md` если менялись токены

**Цель:** Нет утечек, нет компрометации.

---

## ШТРАФЫ ЗА НАРУШЕНИЕ РИТУАЛОВ

| Ритуал | Последствие нарушения |
|---|---|
| 1 (начало) | Работа вслепую, ошибки, рефакторинг назад |
| 2 (конец) | Потеря контекста, повторная работа |
| 3 (pre-commit) | Сломанный main, реверты |
| 4 (pre-deploy) | Падение PROD, инцидент |
| 5 (фичи) | Рассинхронизация плана и реальности |
| 6 (тесты) | Регрессии, неработающие детекторы |
| 7 (обзор) | Застревание, потеря фокуса |
| 8 (безопасность) | Утечка токенов, компрометация |
