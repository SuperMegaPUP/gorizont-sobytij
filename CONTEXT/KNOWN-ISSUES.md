### Средний BSCI ~0.50 (завышен)
- **Статус**: ЧАСТИЧНО ИСПРАВЛЕН
- **Проявление**: BSCI теперь варьируется (0.33-0.68), но среднее всё ещё ~0.50

## Из спецификации v4 — незакрытые баги

### Фильтр мёртвых тикеров
- **Статус**: ЧАСТИЧНО ИСПРАВЛЕН (в TOP-100 фильтрация работает: BSCI>0.03 ИЛИ ≥2 активных детектора)
- **Проблема**: YNDX, POLY и другие с BSCI около 0.27 без реальных данных всё ещё могут попадать в радар
- **Решение**: если все detector scores < 0.15 на протяжении 3+ сессий → excluded_from_radar

### Radar CumDelta=0 не по центру
- **Статус**: НЕ ИСПРАВЛЕН
- **Проблема**: Текущий код использует minCD/maxCD для оси X → ноль не по центру
- **Решение**: absMaxCD = max(abs(minCD), abs(maxCD)), ось от -absMaxCD до +absMaxCD

### Коэффициент нормализации
- **Статус**: НЕ ИСПРАВЛЕН
- **Проблема**: 0.25 в cross-section-normalize.ts даёт слабую дискриминацию
- **Решение**: увеличить до 0.4

## Исправленные

### ~~BSCI=0.00 у ВСЕХ тикеров + TOP-100 пустой + Радар сломан (выходные/ночь)~~ (ИСПРАВЛЕНО 2026-04-26)
- **Статус**: ИСПРАВЛЕН
- **Причина 1**: `canGenerateSignals()` в `scanTicker()` блокировал ВСЁ сканирование когда рынок закрыт (выходные, ночь) — детекторы вообще не запускались
- **Причина 2**: `staleData` шорткат в `scanTicker()` обходил все детекторы когда данные старше 30 мин — в выходные ВСЕ данные протухшие
- **Причина 3**: Агрессивный фильтр `BSCI > 0.03` в TOP-100 и `BSCI > 0.05` в радаре — при BSCI ≈ 0 все тикеры удалялись
- **Исправление**:
  - Удалён `canGenerateSignals()` gate из `scanTicker()` — проверка должна блокировать ТОЛЬКО генерацию сигналов, не детекторы
  - Удалён `staleData` шорткат — детекторы всегда запускаются, даже на протухших данных
  - Убран фильтр BSCI из TOP-100 route — все тикеры показываются
  - Убран фильтр BSCI из радара — ≤50 тикеров: показывать все; >50: smart filter (alerts + futures + top-30)

### ~~ТОП 100 не грузился — только 9 фьючерсов~~ (ИСПРАВЛЕНО 2026-04-26)
- **Статус**: ИСПРАВЛЕН
- **Причина 1**: Сканирование 100 тикеров таймаутилось на Vercel (5 мин лимит)
- **Причина 2**: Динамический fetch тикеров с MOEX не работал как fallback
- **Исправление**:

  - `fastMode`: для TOP-100 пропускаем RVI и FuturesOI (200 вызовов экономии)
  - 4 мин лимит с автоматическим сохранением прогресса
  - Cron jobs: автоматический скан каждые 3 часа в торговые дни
  - Auto-trigger: GET запрос автоматически запускает POST скан в фоне
  - Фильтрация: ~~BSCI > 0.03 ИЛИ ≥2 активных детектора~~ УБРАНА — показываем все тикеры

### ~~BSCI идентичный 0.52 у всех тикеров~~ (ИСПРАВЛЕНО 2026-04-25)
- **Статус**: ИСПРАВЛЕН
- **Причина**: Новые P1 детекторы возвращали ~0.5 fallback значения когда нет trade data
- **Исправление**: DARKMATTER (trades<10 → score=0), DECOHERENCE (trades<20 → score=0), HAWKING (n_trades<50 → score=0)
- **Остаток**: Средний BSCI ~0.50 (завышен) — см. выше

### ~~OFI = 0.0 у ВСЕХ тикеров (оригинальный баг)~~ (ИСПРАВЛЕНО)

- **Дополнительно**: Добавлен Real-time OFI (Cont et al. 2014) через Redis-кеш предыдущего снапшота стакана

## Архитектурные решения

### Детекторы всегда запускаются (даже в выходные/ночь)
- **Принцип**: `scanTicker()` ВСЕГДА запускает все 10 детекторов, даже если рынок закрыт
- `canGenerateSignals()` проверка НЕ должна быть в scanTicker — только в генерации сигналов
- `staleData` флаг информационный — НЕ шорткат для обхода детекторов
- В выходные/ночь: BSCI ≈ 0 (спокойно), все детекторы вернут низкие скоры → GREEN
- Это ПРАВИЛЬНО: нет данных = нет аномалии = GREEN (не 0.5 fallback!)

## Низкие

### TOP-100 сканирование медленное
- **Статус**: УЛУЧШЕНО (было: таймаут; стало: 2-4 мин с инкрементальным кэшированием)
- **Время**: ~2-4 минуты на 100 тикеров (зависит от MOEX API)
- **Кэширование**: Redis TTL 30 мин, инкрементальный прогресс 10 мин
- **Cron**: каждые 3 часа в торговые дни

### Формула уверенности в сканере упрощённая
- **Статус**: ИСПРАВЛЕНО (Спринт 4, Фаза 2)
- **Сейчас**: условное взвешивание BSCI + conv + RSI/CRSI + роботы + дивергенция
- **Файл**: `signal-generator.ts`

### Детекторы требуют доработки по спецификации v4.1
- **DARKMATTER** (П1, ✅ ВЫПОЛНЕНО): expected_entropy + iceberg consecutive + MIN_ICEBERG_VOLUME
- **DECOHERENCE** (П1, ✅ ВЫПОЛНЕНО): символьный поток + tick_rule при ΔP=0
- **HAWKING** (П1, ✅ ВЫПОЛНЕНО): Welch PSD + noise_ratio fix + N≥50
- **BSCI** (П1, ✅ ВЫПОЛНЕНО): η=0.03, min_w=0.04
- **GRAVITON** (П2): экспоненциальная модель → центры масс + walls
- **ACCRETOR** (П2): угловые сектора → DBSCAN
- **CIPHER** (П2): нет z-score перед PCA, нет condition number check
- **ATTRACTOR** (П2): галлюцинации на мёртвых тикерах → stickiness по spread + volume_profile
- **ENTANGLE** (П2): нет ADF-теста стационарности
- **PREDATOR** (П2): нет FALSE_BREAKOUT градиента, нет estimated_stops
- **WAVEFUNCTION** (П2): нет ресэмплинга, нет log-weights → PF вырождается
